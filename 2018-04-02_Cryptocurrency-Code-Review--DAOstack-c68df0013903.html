Cryptocurrency Code Review: DAOstack

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

So, I think it is incredibly early days, and I think these guys are going to market way to early, I also don’t see why a DAO needs it’s own token. With all of those personal bias out of the way, let’s look at the DAO.

The DAO or Decentralized Autonomous Organization is a concept that allows for a decentralized version of classical organizations. So for sake of ease, let’s look at the classical business. You have different supporting functions HR, Finance, Compliance, Legal, Administration, Governance. Normally these functions are executed by different levels of authority in an organization, board of directors, chief executives, managers, and so forth. The DAO’s purpose was to have all of these auxiliary functions exposed and managed as smart contracts and to allow the community involved, a framework from which to do governance (thus ignoring the traditional mechanisms of management). This allows for more open and transparent organizations with autonomous governance.

So the above is fantastic when you think of it in a traditional sense, the shareholder. If you currently own minority shares in a company, your votes mean very little, in fact most of the time, you won’t even be represented. In a DAO your votes have value since each shareholder can form part of the governance, so the DAO is a fantastic model if you put it on top of an existing company, for example if Ethereum was a DAO everyone that held Ether (essentially shares) had a vote and could manage the governance.

In DAOStack, you have stack tokens. I’m missing something here, because I don’t see the governance value proposition unless it was native tokens to an organization. My assumption is that DAOStack Stack tokens is to DAOStack what Ether is to Ethereum, so it is for governance of them? I need to read up more on this, but first let’s jump into the code;

We start off with Arc, we can call this the system core since it allows for the modular selection and configuration of smart contracts to manage your organization.

Lot’s of commits, lot’s of branches, lot’s of contributors. All good signs, let’s see what the code looks like;

So we start off with their main value proposition, pre-built libraries that you can plug and play for your organization, this is what gives them their convenience value after all, so let’s investigate a few.

ContributionReward.sol looks good, good event processing, good standards, good comments. The design and structure is well thought out, a few arrays used which I’m not a fan of because it means you need understanding of the incoming payload, but it’s minor.

They have models that are fueled by DAOToken, native tokens, or Ethereum, that’s good for adoption, although does slightly decrease the value of DAOToken. I like the idea of the fixed non transferable reputation token.

A lot of work has gone into these contracts, they are detailed and well thought out. This team is well versed in smart contracts

DAOToken is ERC827, interesting, ERC827 allows you to embed other function calls in transfer mechanisms, good choice for the tokens purpose.

Reputation token is nice and straight forward, mint or burn, perfect for positive/negative voting schemes.

Their smart contracts are good, let’s move on to their web wrapper, arc.js

Good abstraction, good design patterns.

Web3.js as expected.

Alchemy is fairly straight forward webapp code, express webapp being hosted and reading from web3.js, redux design pattern is nice to see.

Conclusion: Good, complex solidity code. Straight forward, but good webapp code, most of the contracts are done and the webapp wrappers look good. It’s an experienced team. From a pure code perspective I don’t see any issues or red flags, they are probably the most well versed in solidity of any organization I have reviewed so far, there is a lot that can be learned here.

I personally think it’s too early for this kind of product or model, blockchain itself is still too young, so I think they will struggle with adoption. I also wonder how much real value the token itself will have since organizations could just instead use their boilerplate and use ETH. The underlying constraints of ETH transaction fee’s will also still remain a problem. Personally I would have gone with a side chain implementation that’s rooted in Ethereum.

Great Solidity code though.
