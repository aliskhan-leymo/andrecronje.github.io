Cryptocurrency Code Review: Celsius

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

Let’s start from the bottom and work our way up.

I really like their technology stack, React-Native, Redux, and then Web3.js, so we can expect a mobile app that interacts with Ethereum. This already intrigues me because I expect to see some form of a wallet then, but I’m not seeing the native modules to support wallet building. This is an utter horrible thing to try and do, I’ve done it, it was painful.

But let’s see if we can find it in

I see they at least have crypto-js, but that won’t be enough, let’s see what we are working with. This is still an Expo app, they have yet to unpack it, this tells me they are still incredibly early days since they haven’t gone through the native library hardships yet, but let’s reserve that comment for later and keep going with our investigation.

Boiler plate react and redux so far, nothing interesting to report.

I’m hoping that comment is just a typo and that the password isn’t infact just the plain user password but instead a joined hash with a salt.

Standard actions with json-rpc. They are using a mix of REST and JSON-RPC, whatever works I guess.

Now we are using parameterized URL’s, stick to one pattern guys.

Looks like they are wrapping the main ethereum functions in their API, I don’t think that it’s published, also interested to see what getSecureStoreKey looks like.

More boiler plate redux.

VerifyPhoto doesn’t actually do anything
VerifyPhoneNumber doesn’t actually do anything
VerifyDocument doesn’t actually do anything

You get the point

This is all fake boilerplate for now, they haven’t done any of the meat yet, let’s move on

Guys, look at that instruction, — rpccorsdomain ‘*’, this opens up your node to everyone that can access it, never use this when hosting a domain. I like what they were trying to do with this passport proxy, but please don’t follow these instructions

Express boilerplate. Just more boiler plate guys, nothing here.

Let’s move on to the celsius app instead. After going through a few screens I stopped, there is nothing here.

Conclusion: This is just a boiler plate react app, they haven’t done anything with Crypto yet, no wallet creation, no mnemonic, no bip39, nothing. It’s just a few pretty screens talking to a boiler plate REST/JSON-RPC backend with a basic nod, but severe lack of any real security.

They still have all the hard work to do, they have done the incredibly easy (talking 2–3 weeks effort) part, they haven’t done any of the actual Crypto. Encryption or Security side of things (which I thought was the core)

This is very basic, HalloWorld react-native-redux stuff.
