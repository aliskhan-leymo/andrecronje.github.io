Cryptocurrency Code Review: Nebulas

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

Lot’s of activity and development, lot’s of contributors, there is a definite Ethereum inspiration here but it is very far removed from it.

Using ProtoBuff, always happy to see that.

Own stack and sort implementation, I wonder why.

They are working on their own DAG?

Cheeky, I like it.

Nothing here, yet.

No substance in consensus PB, yet.

Built in fork consensus, good to see.

Remote signing from NewAdminServiceClient, interesting.

Dynasty for delegates kept in its own Trie.

ParallelNum, setup for parallel chains.

More implementation skeletons for parallel chains. They are planning ahead.

Blocks can revert, but there is set finality.

Good abstraction of core blockchain values in transaction, means forking and deploying private chains will be easier to configure.

All transaction execution is compartmentalized, it’s nice to see.

No ranking structure just yet.

At least they added the comment that they are planning on adding a secure connection.

So far, it’s good code, I don’t have any comments or concerns, they have all the default components well designed, the architecture is good, the encapsulation and abstraction is very well thought out and implemented. I dare even say I might like their layout a bit more than go-ethereum, definitely for their transaction layout.

But now let’s consider what makes them unique

This doesn’t exist, yet. From what I have seen they have proven that they have the technical skillset to implement it, but for now, they are just another carbon copy blockchain. An accomplishment in it’s own right, but these are problems that have already been solved.

Let’s move on to neb.js

The web3.js equivalent, less active but still substantial repo, let’s dive in.

Fantastic comments and documentation, this goes a long way. Again, very little to say unfortunately, it’s good.

Conclusion: Good code, all the standard blockchain 101 components have been ticked off, I can see architecture has been created for future enhancements and some of the foundations have already been created. This is a good sign of a long term roadmap.

Right now though, there is nothing that differentiates them from the other players, while the foundations are there, they still have a lot to build, all their key value propositions are currently missing, but they have shown a code level competence that should give people confidence that they can achieve and fulfill on their roadmap. In fact I think they should be ready before Q4 given their current progress.

They still have all the hard puzzles to solve, but it looks like they have the right team to solve it.
