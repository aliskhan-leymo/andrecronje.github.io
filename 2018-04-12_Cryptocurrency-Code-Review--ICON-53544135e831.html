Cryptocurrency Code Review: ICON

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

Icon, the network of networks. Essentially an interconnected blockchain ecosystem, what Wanchain/Aion/Fusion are doing for the Ethereum ecosystem, Icon is doing for themselves (while partnering with Wanchain and Aion?)

So, what I’m excited about is the interconnected aspect of the distributed blockchains to the ICON Nexus.

I start off with searching github for ICON and I find

39 commits behind, last commit August 31, 2017. Surely this can’t be the right repo?

I go to coinmarketcap and go to their github link.

Last commit Feb 21, this can’t be it either. We head over to the Icon website, and go to their github link.

I guess this is it? Very confusing guys, can we please stick to one repo? Let’s begin.

Last commit Feb 21, almost 2 months ago. Guess we are reviewing this then.

Only 57 commits, only 2 branches, and 12 contributors.

GRCP and Protobuff for communication. levelDB for storage. Flask for HTTP. (These are just all technologies that are being leveraged, you don’t have to worry too much about them)

Very dangerous assumption here, the application has to be run as root, and while they are at it they add their own entry into known_hosts.

Basic little stand alone key creator, nothing interesting.

Basic standalone CA cert creator, noting interesting.

Nothing in SCORE (Smart Contract On Reliable Environment).

Some empty implementation files.

Looks like we finally found the core.

Empty test files are usually not a good start.

Not what I was expecting. Guess ChainHelper hasn’t been implemented yet.

SCORE helper just has; load database, create database, and logging. Not sure how it’s helping SCORE.

At this point I’m assuming this is the wrong repository or they aren’t open source. But let’s continue.

Configurable consensus, that’s good to see.

Consensus kicks off as soon as a block is received.

Voting is straight forward, receive an unconfirmed block, validate it, if validated try to add to the unconfirmed blocks, if it hasn’t been confirmed yet cast a vote that it is a valid block.

All of this is happening, on a code level, on the peer service. Expected a bit more abstraction and encapsulation, let’s assume this is still very much proof of concept.

I can’t seem to find any of their SCORE implementation.

Transaction signing and verification, straight forward. Voting and peer discovery followed a similar pattern.

Conclusion: It’s a weird mix of production level code and proof of concept code, it doesn’t feel like the project was written by the same team. It’s difficult to say if it’s good or bad, because in honesty, it is both. Some sections are very well written, others seem like it was rushed together by an intern.

I couldn’t find their SCORE implementation, just a lot of stubs showing that it will still be implemented. All the rest are present, p2p, block, transaction, validation, transaction packing, channels, and consensus, so it is by definition a blockchain. But it’s just a blockchain. Just a blockchain that hasn’t been updated in 2 months.

Where is the first class citizen cross chain implementation?
