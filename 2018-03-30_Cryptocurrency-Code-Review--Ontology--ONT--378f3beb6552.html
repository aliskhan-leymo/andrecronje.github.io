Cryptocurrency Code Review: Ontology (ONT)

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

Everyone seems very excited about Ontology (you guys can stop spamming my channel now please). Not really sure where to start, there is a lot going on here, we have ontology-ts-sdk (typescript library), we have ontology (the big boy, think we will start here), we have a java sdk, an eventbus, wasm, and even an python oracle. So a lot to cover, let’s start with ontology though.

Active repo, lots of contributors, lots of commits. Readme looks good, let’s jump in.

So, code looks familiar. This is DNA Project’s code (which is also NKN project code, which is also Elastos code). Let’s see what they have done extra though.

Ok, so it looks like tanZiWen is now working for Ontology? He did a lot of the DNA project work, so that’s a good sign. It looks like DNA Project split and has become NKN and Ontology, don’t know what happened with the original project, but let’s ignore any potential politics for now.

Does look like they added quite a few consensus models.

VBFT not available yet.

SBFT not available yet. Solo is just for self hosted solutions. So for now it’s just dbft.

I like that they added a bunch of different key encryption schemes, makes it a nice codebase to inherit from, but it doesn’t directly add a lot of value.

For their WASM implementation they used Wagon, so skipping for now, for their NEOVM it is just a slightly modified version of AVM, so skipping as well for now.

Conclusion: So, the rest of the code was fairly boilerplate, had a dive into validator, txnpool, vm, smartcontract, but nothing really to report. Nothing bad, but nothing great, they know the DNA project code base very well and they have made some improvements. I have yet to see the codebase in production though, so it’s difficult to say how it will perform. For now it’s still very early days but no red flags here, they have the core, but they have yet to bring all the auxiliaries (cross-chain, high performance, horizontally scalable) to the table.

I’ll keep having a look a the repo every so often, and I’ll dive some of the auxiliary projects as well.
