Cryptocurrency Code Review: CyberMiles

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

100 times faster then Ethereum.

Repo is a bit confusing, there is a lot going on here. Travis looks like the blockchain, so let’s keep it in mind, the go-ethereum fork doesn’t have much happening, same with the tendermint fork, ethermint looks empty, ok this leaves us with travis-java, VM, and Travis.

Let’s grab travis

Travis uses the Cosmos and Tendermint SDK’s, tendermint is a Byzantine fault-tolerant replicated state machine, and Cosmos is the decentralized network of parallel Tendermint chains. So I guess we can say that CyberMiles is just another chain in this eco system.

So a bit of a Frankenstein going on here, we have Cosmos for cross-chain, we have Tendermint for BFT consensus and we use Ethereum standards for transactions. To be honest, I like it, and what is more amazing is that they are all working together.

And this shows the amalgamation, Cosmos used to load the transaction, ethereum used to decode it, ethermint used to deliver it, this is kind of beautiful, in a monster kind of way.

At this point I became a bit skeptical, and it started looking a lot like Ethermint, so I did a quick comparison

On the right, Ethermint, left Travis, so it’s definitely, let’s say, inspired by Ethermint, I think there is still enough going on here though to say that they are standing on their own two feet, so let’s keep going, but we have to keep this in mind to separate what they have done and what Ethermint did.

Validator key’s published in blocks based on their dPoS implementation.

So, not a huge fan of a direct DB implementation, disk access is notoriously slow, but the great news here is that it still leaves room for improvement if they shift this to a memory store.

Conclusion: To be fair, they aren’t doing the heavy lifting, they are letting Tendermint, Ethermint, Ethereum, and Cosmos do the heavy lifting. That being said, it’s implemented and joined in a really novel way, this isn’t easy to do, and it actually means that they can leverage the best parts of each of those systems.

It’s a very interesting design pattern, but I really like it, will definitely keep an eye on them. Surprised they aren’t bigger.
