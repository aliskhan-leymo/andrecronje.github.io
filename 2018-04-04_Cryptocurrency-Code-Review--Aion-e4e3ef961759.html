Cryptocurrency Code Review: Aion

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

Lot’s of commits, great branches usage (per feature set), lot’s of contributors.

Design wise, fantastic architecture, everything is designed as their own self standing module, easy to pull them apart.

Little sparse on the comment side, but luckily we don’t need comments.

So, modular, and optimized are a bit of a double edged sword. Everything here is design to be plug and play and config driven, which is fantastic from a implementation point of view, but sadly, not so great from an optimization point of view. Config’s need to be checked a lot and this does cause overhead. That’s the thing with coding at the end of the day, each choice is just a trade off.

The type choices here confuse me a bit, these are final values, some of these do not need to be long. Again, not super optimized, but not bad.

Error handling could be done a bit better.

Conclusion: When I nitpick about such small things, then you know the code is good, I like the architecture, I like the design patterns, the files are kept neat and bite size to make each module easily understandable. I was somehow expecting more code, but quantity isn’t a descriptor of quality. This is just good code.

The one thing I am sad about though, is that I don’t see their cross-chain protocol, and that as far as I’m concerned is the thing that differentiates them, I feel a lot of time, focus, and energy is going into catching up with the current blockchain game, I really wished they would have skipped that and just focused on their cross-chain implementation.

Aion right now is supposed to be the cross-chain leader, but I’m not seeing any cross-chain. They are also supposed to be blockchain 3.0, but from what I can see they are still on 1.0

They still have a lot of catching up to do before they can start running ahead, I really hope we start seeing some cross-chain implementations soon, otherwise they might lose this game. It’s good code though.
