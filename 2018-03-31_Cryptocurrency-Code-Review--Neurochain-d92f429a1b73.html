Cryptocurrency Code Review: Neurochain

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

Prototype looks like the real meat of the Proof of Concept (PoC), so let’s have a look

Admittedly, this is just a demo, their real project will be in C++, so we can’t judge too much based on what we see

NodeJS has two variable types, var (the object will change) and const (the object is fixed), the latter is more optimized, I like seeing const used instead of var, but again, this is a PoC, so we can’t expect high quality.

A bot is made for each purpose, this is a microservices model for bots, I like it.

The bad, using global as a shared pattern, the good, fantastic comments.

This is the meat, let’s have a look;

Websockets for activity.

Transactions have a reference to lastBlockID, interesting.

All the basics covered, hash and signature.

No mining yet, but transactions, signatures, and messages implemented, let’s keep going

Consensus creates a quorum based on bot entropy and then selects a bot to propose. Nice mechanism.

Conclusion: For now the machine learning is just a gimmick, but I like the bot swarm micoservices design architecture. I don’t know why they want to re-write it to C++, they should keep building on this base, it’s a nice design and it’s nice code.

For now it’s just messages being passed around and an elected bot creating a block, content in messages is left variable, that’s a nice mutable implementation. I like the design architecture, will keep an eye on them.
