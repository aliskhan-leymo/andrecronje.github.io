Cryptocurrency Code Review: Holochain

Disclaimer: These reviews are done as is from what is on display in the master branch of the repo’s made available. This review is not a comment on the overall project, scope, or success thereof. This was done as an educational review by me and any comments in the article are simply my opinion. It should not be used as any comment or advice on the project as a whole.

Let’s start off with, what is HoloChain, in the traditional sense, HoloChain is not a blockchain, it’s a new kind of distributed internet. The idea focuses around agent centric consensus. A traditional blockchain is data centric consensus, nodes reach consensus about what data they deem to be valid and accepted. Agent centric allows the users to decide on their consensus.

HoloChains have two parts, the original hashchain and the shared Distributed Hash Table. First, the hashchain, each app is uniquely identified by their root entry in their hashchain, only hashchains that share this root entry are considered shared, these can then communicate with other hashchains with the same starting root. This is comparable to git (a source control management protocol), we all start from the same root, and then we commit our changes into the project, sometimes this causes conflicts, and then we agree on resolution (in this case conflicts aren’t permitted in the first case).

The idea here, is that not everyone needs awareness of what everyone else is doing. If we are having a chat on a distributed chat client, does everyone have to agree on what we are saying? Probably not, in this case, we are all using the app, but only the two of us are committing to each others hashchain. If we wanted to make it public, we could share it with the Distributed Hash Table, but for most applications this most likely won’t be required.

So if this is the case, how can we accomplish balance transfer in the traditional sense? Our roots are embedded into the DHT in this case, so that everyone can share on the given state of the overall system, when we mutually agree that transfer has happened between the two of us, we can then publish this to the DHT. So the same as the chat example, except now we start from a shared root.

This is a framework for decentralization, could you build a blockchain on it? Sure, could you also build a high performance social network? Yes, yes you could.
