Open Source, Not Open Source

This post is really just a rambling of thoughts and ideas and may seem a bit incoherent at times.

Let me start by explaining why I do these “reviews”, I put it in quotes because my intent was never to do reviews, but instead to use this as a teaching mechanism for myself. A writer will tell you that the best way to become a better writer is to read a lot. Similarly I believe the best way to improve your coding skills is to read a lot of code, so this is what I do. In the crypto space, we have the whitepaper, which to a developer is really just a teaser of things to come, so often I am filled with such excitement and anticipation after reading a whitepaper that I rush to the Open Source project’s code base to try and consume everything they have on offer. Sadly, in the Crypto space we have developed the Not Open Source, this isn’t closed Source, this is just where we act like it’s Open Source but in reality it is just a smokescreen.

So how this this conversation start, I had a great discussion with @Grandbear yesterday (Q from EximChain) with regards to the current state of Open Source, he very appropriately pointed out that a lot of my reviews come off as scathing and that their purpose is to create FUD (Fear Uncertainty Doubt) when we should be focusing on building each other up as every project contributing opensource code is crucial this early in the space.

This made me realize I need to take a bit of a step back and explain myself. I have been a salaried or consulting developer & architect most of my life, 99.9% of everything I have ever written is closed source and proprietary. Some of the code that I am most proud of to this day, I can show to no one. This weights heavy on my heart.

So, why is Open Source so important to me? The times that we had the greatest technological advancements were during times of war, what was different then? The difference was we were banded together and working towards a common goal. Organizations that would normally keep code proprietary or closed source would willingly share their technology with each other for the greater good instead of personal gain. This I believe is the vision of Open Source, not just transparency, but the community begging that we please band together and help improve each other.

So at this point, there are a few categories of Open Source that I would like to discuss;

First is the traditional idea, that a set of code is made publicly available and the community can contribute to it via PR’s (Pull Requests). Here we have the full transparency Open Source teams, Ethereum, Aion, RChain, Zilliqa, ChainLink, and there are many other great examples, these are the standards of Open Source for me and I have learned so much from each one of them, a sincere thank you.

The second is the fork, this allows me to copy another set of code, while keeping the original line of ownership very clear, this can have great cross community functionality if the one fork wants to deviate, however both still keep core mechanisms the same, this allows for cross-PR’s where both communities can contribute to each other. A great example here is Zcash which started out as a Bitcoin fork.

Then we have the clone, this is normally where another code base was cloned with the intent to develop on it but not necessarily to contribute back to the original base. This however keeps the history in tact, which is a friendly nod towards the original developers.

Lastly we have the copy, development history is wiped, there is no link to the original fork and there is no nod to the original community.

Which one sounds more awesome?

Now, let’s talk about Not Open Source, we have the halve open source, the examples I want to use here are EximChain, and TomoChain, EximChain on the one hand have this amazing piece of software, their terraformer, which allows you to boot up nodes across multiple zones and it is service agnostic, this is an amazing tool that I can see being adopted by a lot of companies, I myself have forked it and intend to use it in the future after making some modifications to support Hyperledger Fabric and if I can do this in an agnostic way I’ll be sending my PR back.

Then on the other hand they have the fork and rename.

TomoChain have their example cross-chain app which I really like and also did a fork from, and then they have the fork and rename.

So fantastic code on one end, and a smokescreen on the other, but, this is our own fault as a community, because if they didn’t have the smokescreen we would have been complaining about how they are closed source.

To measure companies we have essentially created a manipulable checklist

  • Token metrics? Check
  • LinkedIn team? Check
  • LinkedIn advisors? Check
  • Social media? Check
  • Open Source? Check…

And we publicly publicize that this is how we measure, so what should a company do? They have a few options

  1. Just admit it’s closed source and suffer the community backlash
  2. Put everything open source and risk competitors stealing it
  3. Put up a fake smokescreen and develop behind the scenes

And we live in a capitalist society, it’s not like I’m sitting at my day job working for friendship and happiness, so there is an incentive to keep source code closed to protect IP, so now we are seeing the patent space a lot, closed source until patent registered, but that is a conversation for another day.

So what’s the big problem with this? My issue, how do you differentiate the real deal from the frauds? We have Crypto 100 (Top 100 coinmarketcap) companies who’s githubs look exactly like scam coin githubs. For example, I received feedback from WanChain and TomoChain that their work is happening closed source and that their githubs are just for public presence. By their own admission their public githubs are absolutely pointless (and thus so are my reviews) and just there for the tickbox above?

And this is actually the thing that got me so frustrated in this space, specifically I remember the day I finished reading Wanchain’s whitepaper and how excited I was to go through their code, even moreso when I found out they are a go-ethereum fork, because I have been working extensively on go-ethereum which means there is a lot I could learn, and when I started going through commit after commit after commit with just name changes. This was disheartening to me.

This by no means implies that they won’t be able to deliver absolutely amazing projects, but unfortunately it also means we can’t differentiate between the fakes and smokescreens.

How do we tell the difference between the company that just cloned geth (go-ethereum) renamed it and launched their mainnet vs the company that is actually doing work?

But again, why is Open Source so important? So right now, there are about 8 highly recognized companies all building on geth, they will each make advancements and discoveries, but none of them will be shared, because each one is protected by IP. Doesn’t that sound incredibly sad? Now imagine if they were truly open source and each one contributed together to elevate it to the next level! We are at the precipice here of a technological revolution, but instead of working together, we are dividing our communities more and more until we are left with exactly what we have now.

So please, I’m begging all enthusiasts, let’s start being critical of real enhancement and let’s start paying close attention to these Not Open Source companies and mainnets. This doesn’t mean that we won’t see great things from these teams, it just means right now, we can’t even differentiate the real deal from the scam.
